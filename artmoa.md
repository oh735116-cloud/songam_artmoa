# 공연 정보 통합 웹앱 UX/UI 및 GAS 구현 기획서

> **과제명:** 공연 정보를 한 번에 모아볼 수 있는 반응형 GAS 웹앱 제작  
> **평가 주제:** 능력단위평가  
> **제출 기한:** 2026년 6월 10일  
> **제출 폴더명:** `화면구현_이름`  
> **프로젝트 형태:** UX/UI 설계 + Figma 반응형 디자인 + Google Apps Script 웹앱 구현

---

## 1. 프로젝트 개요

### 1-1. 주제

**공연 정보를 한 번에 모아보고, 지역·연령층·장르별 맞춤 추천과 검색까지 제공하는 공연 정보 통합 웹앱**

사용자가 여러 사이트를 따로 방문하지 않고도 공연명, 장르, 지역, 날짜, 장소, 가격, 예매 링크 등을 한 화면에서 확인할 수 있도록 돕는 반응형 웹앱이다.

### 1-2. 서비스명

**공연모아**  
영문명: **ShowMoA**

### 1-3. 한 줄 소개

> 흩어진 공연 정보를 한곳에 모아 사용자의 지역, 연령층, 관심 장르에 맞는 공연을 추천하고 검색·저장까지 지원하는 공연 정보 큐레이션 웹앱

## 2. 기획 의도

### 2-1. 문제 인식

공연 정보를 찾기 위해서는 공연장 홈페이지, 예매 사이트, 공공데이터 사이트, 지역 문화포털 등을 각각 확인해야 하는 불편함이 있다. 특히 사용자는 다음과 같은 어려움을 겪는다.

| 문제점 | 설명 |
|---|---|
| 정보 분산 | 공연 정보가 여러 사이트에 흩어져 있어 비교가 어렵다. |
| 검색 피로도 | 날짜, 지역, 장르별로 원하는 공연을 찾기까지 시간이 오래 걸린다. |
| 모바일 불편 | 일부 공연 정보 페이지는 모바일에서 보기 불편하다. |
| 관심 공연 관리 어려움 | 나중에 보고 싶은 공연을 따로 저장하기 어렵다. |
| 일정 확인 불편 | 공연 날짜와 장소를 한눈에 확인하기 어렵다. |
| 맞춤 추천 부족 | 사용자의 지역, 나이대, 관심 장르에 맞는 공연을 자동으로 추천받기 어렵다. |

### 2-2. 해결 방향

공연 정보를 카드형 UI로 정리하고, 필터 검색과 관심 공연 저장 기능을 제공하여 사용자가 공연 정보를 빠르게 탐색할 수 있도록 한다. 여기에 **지역별, 연령층별, 공연 장르별 맞춤 추천 기능**을 추가하여 사용자가 직접 모든 정보를 찾지 않아도 자신에게 맞는 공연을 빠르게 발견할 수 있게 한다. 또한 GAS와 구글 스프레드시트를 활용해 관심 공연 저장, 문의 기록, 추천 조건 저장, 자동 알림 기능을 구현한다.

### 6-1. 맞춤 추천 기능 상세 설계

사용자는 처음 접속했을 때 또는 검색 영역에서 자신의 조건을 선택할 수 있다. 웹앱은 선택된 조건을 기준으로 공연 데이터를 분류하고, 추천 점수가 높은 공연을 상단에 보여준다.

| 추천 기준 | 선택 예시 | 추천 방식 |
|---|---|---|
| 지역 | 서울, 대구, 부산, 광주, 경기 등 | 사용자가 선택한 지역과 공연 장소가 일치하면 우선 추천 |
| 연령층 | 10대, 20대, 30대, 가족, 중장년층 | 공연 성격과 가격, 장르 태그를 기준으로 추천 |
| 장르 | 클래식, 뮤지컬, 연극, 콘서트, 국악, 무용 | 선택한 관심 장르와 일치하는 공연 우선 노출 |
| 가격 | 무료, 1만 원 이하, 3만 원 이하 | 학생·청년층에게 무료/저가 공연 우선 추천 |
| 날짜 | 오늘, 이번 주말, 이번 달 | 가까운 날짜의 공연을 우선 추천 |

### 6-2. 추천 알고리즘 예시

복잡한 AI 알고리즘 대신 GAS 과제 수준에서 구현 가능한 **점수 기반 추천 방식**을 사용한다.

```text
추천 점수 = 지역 일치 점수 + 연령층 적합 점수 + 장르 일치 점수 + 날짜 근접 점수 + 가격 적합 점수
```

| 조건 | 점수 예시 |
|---|---:|
| 사용자가 선택한 지역과 공연 지역이 같음 | +30점 |
| 관심 장르와 공연 장르가 같음 | +30점 |
| 연령층 추천 태그와 일치함 | +20점 |
| 이번 주 또는 이번 달 공연임 | +10점 |
| 무료 또는 저가 공연임 | +10점 |

최종적으로 점수가 높은 순서대로 `오늘의 추천 공연`, `내 지역 추천`, `내 취향 추천` 영역에 표시한다.

### 6-3. 검색 기능 상세 설계

검색 기능은 사용자가 직접 원하는 공연을 찾을 수 있도록 제공한다. 추천 기능이 자동 탐색이라면, 검색 기능은 사용자가 직접 조건을 입력하는 탐색 기능이다.

| 검색 항목 | 예시 |
|---|---|
| 공연명 검색 | “레미제라블”, “클래식”, “청춘마이크” |
| 지역 검색 | “대구”, “서울”, “부산” |
| 공연장 검색 | “예술의전당”, “대구콘서트하우스” |
| 장르 검색 | “뮤지컬”, “클래식”, “연극” |
| 복합 검색 | “대구 클래식 무료”, “이번 주말 뮤지컬” |

## 7. 외부 API 연동 계획

### 7-1. 사용 후보 API

| API명 | 제공 정보 | 활용 목적 |
|---|---|---|
| KOPIS 공연예술통합전산망 Open API | 공연 목록, 공연 상세, 공연시설 정보 등 | 전국 공연 정보 조회 |
| 공공데이터포털 문화예술공연 통합 API | 공연 제목, 분야, 기간, 장소, 금액, 이미지 등 | 공연 카드 데이터 구성 |
| 서울시 문화행사 정보 API | 서울 지역 문화행사, 날짜, 장소, 요금 등 | 지역 행사 정보 보완 |

### 7-2. 우선 사용 API

**1순위: KOPIS 공연예술통합전산망 Open API**

선정 이유:

1. 공연예술 분야에 특화된 데이터이다.
2. 공연명, 공연장, 장르, 기간 등 웹앱에 필요한 핵심 데이터가 포함되어 있다.
3. GAS의 `UrlFetchApp.fetch()`로 API 데이터를 가져와 화면에 표시할 수 있다.

### 7-3. API 연동 흐름

```text
사용자 검색/필터 선택
        ↓
클라이언트 JavaScript에서 google.script.run 호출
        ↓
Code.gs에서 UrlFetchApp.fetch()로 외부 API 요청
        ↓
응답 XML 또는 JSON 데이터 파싱
        ↓
필요 데이터만 정리하여 index.html로 전달
        ↓
공연 카드 UI로 화면 출력
```

### 20-2. 가장 추천하는 추가 기능

**가장 추천하는 기능은 `오늘의 맞춤 추천 TOP 5`이다.**

이유는 다음과 같다.

1. 단순한 공연 목록 웹앱보다 개인화 서비스처럼 보인다.
2. 지역, 연령층, 장르별 분류 기능을 자연스럽게 보여줄 수 있다.
3. GAS와 Google Sheets를 활용한 추천 조건 저장 기능과 연결하기 좋다.
4. Figma와 PPT에서 화면 구성을 설명하기 쉽다.
5. 평가 항목 중 UX 설계, 외부 API 연동, DB 연동, 자동화 요소를 모두 강조할 수 있다.

## 22. 자체 평가

### 22-1. 잘한 점

1. 공연 정보를 카드형 UI로 정리하여 사용자가 쉽게 이해할 수 있도록 설계하였다.
2. 모바일, 태블릿, PC 환경을 고려하여 반응형 레이아웃을 계획하였다.
3. GAS, Google Sheets, 외부 API, 트리거를 모두 포함하여 평가 조건을 충족하도록 구성하였다.
4. 단순한 정보 제공을 넘어 관심 공연 저장과 자동 알림 기능을 추가하였다.

### 22-2. 보완할 점

1. API 데이터 구조가 복잡할 수 있으므로 필요한 데이터만 추출하는 과정이 필요하다.
2. 실제 공연 포스터 이미지가 없는 경우 대체 이미지를 준비해야 한다.
3. API 호출 제한이 있을 수 있으므로 로그 기록과 캐싱 전략을 고려해야 한다.
4. 알림 기능은 이메일 발송 권한 승인 과정이 필요할 수 있다.

### 22-3. 느낀 점

이번 프로젝트를 통해 단순히 화면을 예쁘게 만드는 것뿐만 아니라 사용자가 실제로 겪는 불편함을 해결하는 UX 설계가 중요하다는 것을 알게 되었다. 또한 GAS를 활용하면 구글 스프레드시트를 데이터베이스처럼 사용할 수 있고, 외부 API와 트리거를 연결하여 자동화된 웹앱을 만들 수 있다는 점을 배웠다.
